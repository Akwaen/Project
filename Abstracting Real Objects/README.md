The Truck and Engine Management System is a program that helps you manage trucks and their engines. The main goal is to let users create and modify Truck objects, which can be connected to different types of Engine. Users can expect a simple way to work with these vehicles, and the program has built-in checks to make sure all data is correct. What makes this program special is its focus on object-oriented design, which makes it easy to add new features later.

Key features of this program include:
- Create and manage trucks with different attributes.
- Define various engine types with checks to ensure valid inputs.
- Easily get and update truck and engine details using simple methods.
- Clear logs of truck information through a handy toString() method.

The program is organized into two main classes: Truck and Engine. The Truck class includes important details like brand, color, number of seats, towing capacity, and a reference to an Engine object. The Engine class focuses on the type of engine used.

The Truck class connects with the Engine class by using an engine instance, letting truck objects access and change engine details. Data is stored in the class's attributes. The design draws on the principles of object-oriented programming, making everything easy to understand and use.
